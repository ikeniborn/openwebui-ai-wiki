2. Схемы решения и зависимостей

Схема решения (data/control flow)

SP1 data dir (shared volume) · chunks/<domain>.jsonl — records {page_id, domain, kind, embed_text, hash}
▼ inotify (debounced)
SP2 sync sidecar · build_desired → desired set keyed by hash; diff vs state/sync_<collection>.json (hash→entry id)
▼ add(text,meta) / delete(id)
OpenWebUI Knowledge API · upload file <hash>.md → attach to collection; remove+delete on prune
▼ OpenWebUI embeds each entry
LiteLLM /v1/embeddings → bge-m3 (1024-dim, Russian corpus)
▼ indexed
Knowledge collection «ai-wiki» → consumed by SP3 RAG (out of scope)
